8-bit Computer

June 2024
Watch on YouTube
View project on GitHub

Earlier this year, I came back from a camping trip with an inspiration to learn electronics. I started reading a text book, Practical Electronics for Inventors, and began accruing equipment. After building some basic frequency filters, I had the desire to tackle a substantial project. For many years, I've dreamed about building a working computer from just logic gates. I ordered some breadboards, jumper cables, and integrated circuits and set to work.

The integrated circuits I used come from the 7400 series (a family of integrated circuits that perform basic boolean logic functions). Specifically, I used only the AND, OR, NOT, NAND, and XOR versions of these circuits. These logic gates are the basic building blocks of digital logic. With enough logic gates you can replicate any computer in the world. The computer I built, on the other hand, used only about 800.

I cheated in two places. For both the program memory and the RAM I used flash memory chips. This means I can't say I wired the entire computer by hand. While this pains me to admit, I wasn't willing to invest the time or money to remedy it.

Architecture

I built the computer without any formal plans. The reason this is possible is because of modularity. I built a ten modules that each perform a simple task, and then integrated them into the final computer.

You can watch the linked YouTube video for more details about the functionality of each module.

Of course, integrating the modules was the truly hard part. Especially since I didn't bother documenting the inputs and outputs of each module, leading me to reverse engineer my own work. Besides that, I also spent many hours diagnosing mis-wired and faulty connections.

Noise was a big problem. The clock design I came up with was very sensative. This meant that a transient voltage spike/drop would often cause an unintentional clock step. A couple of times I had problems that were solved by simply adding capacitors to make my power supply more stable. I also discovered very late into the build that my clock only works with my oscilloscope plugged into the wall. It doesn't need to be connected to anything, or even turned on. Just plugged into the wall. The moment I unplug it my computer goes haywire.

Machine Language

The assembly language I designed compiles into 24-bit machine language. While the rest of the computer uses 8-bit busses, I decided to splurge on the machine language for two reasons. First, it allowed me to implement a richer set of instructions. Second, it made implementing the instructions simpler. Many of the bits in each instruction have just a single purpose. For example, there are 8 bits that are devoted entirely to selecting the inputs and outputs of an operation, even though some instructions only use 2 bits. Single-purpose bits meant I needed less decoding logic in the computer. I could just run a wire from the bit to the one operation it performed.

If you want to learn more about the assembly language, you can view the README on my Github repository for the computer. In that repository, there are also some example programs that I've run on the computer.

Capabilities

It's reasonable to wonder what this computer is capable of. Of course, the answer isn't terribly impressive. With only 256 lines of program memory, and 256 bytes of RAM it's not possible to perform sophisticated algorithms. That said, with a clock speed of 0.5 MHz it can perform these relatively simple tasks much faster than by hand. The absurd power of modern computers is the only reason this one seems useless. A pre-transistor mathematician would have found plenty of uses for this machine.

Regardless, performance wasn't one of the goals of this project. The actual point was to enrich my understanding of computers, and it did so tremendously.
